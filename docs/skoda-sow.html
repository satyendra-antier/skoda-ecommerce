03
Payments
Payment Gateway Integration ‚Äî BillDesk

Agency to implement secure server-side integration with BillDesk. Client is already onboarded with BillDesk. Integration to be completed for both Test and Production environments.

Integration Requirements
  • Server-side generation of unique Order ID
  • Secure hash generation using Merchant credentials (server-side only)
  • Redirect to BillDesk hosted payment page
  • Implementation of callback URL handling
  • Server-side validation of payment response
  • Automatic order status update based on payment result
  • No card or sensitive payment data to be stored on the server

04
Operations
Inventory Management ‚Äî Centralised Model
4.1 ‚Äî Product Inventory Fields
FieldProduct ID
FieldSKU
FieldPrice
FieldAvailable Stock (integer)
FieldStatus (Active / Out of Stock)
4.2 ‚Äî Inventory Logic
  • Stock to be deducted only after confirmed successful payment
  • Deduction to reflect exact quantity purchased
  • Oversell protection to prevent negative stock at all times
  • Stock validation to run before checkout can be completed
4.3 ‚Äî Out-of-Stock Behaviour
  • Product automatically marked as "Out of Stock" when stock reaches zero
  • Add to Cart button disabled for out-of-stock products
  • Checkout prevented if stock depletes between cart and payment
